# PRD: 범용 DART 정보 검색기 (지역·매출액·업종 조건 기반 거래처 발굴 도구)

## 1. 배경 및 목적

**지역/매출액 수준/업종 등 조건을 입력하면 해당 조건에 맞는 외부감사대상
법인의 요약 재무정보 및 기본정보를 추출해주는 범용 도구**를 만든다.

- **현재 단계(Phase 1) 범위: 데이터 수집기 개발에만 집중한다.**
- 전단지 생성 등 활용 단계(Phase 2)는 이후 별도로 착수하되, Phase 1에서
  수집하는 데이터 필드는 Phase 2에서 필요한 항목까지 미리 고려하여
  설계한다 (나중에 재수집하지 않도록).

## 2. 범위 (Phase 1: 데이터 수집기)

### 포함
- 조건 입력값: **지역**(예: 경남 전체 또는 시/군 단위), **매출액 범위**
  (예: 60억~150억), **업종**(DART 표준산업분류/업종코드 기준, 복수 선택 가능)
- 조건에 맞는 외부감사대상 법인 목록 자동 추출 (OpenDART API 기반)
- 회사별 기본정보 및 요약 재무정보(당기·전기) 수집 → 3장 참고
- 결과를 CSV/Excel로 산출 (조건별 반복 실행 가능한 구조)

### 제외 (Phase 1 범위 아님)
- 전단지/진단자료 자동 생성 (Phase 2, 추후 별도 진행)
- dart.fss.or.kr 화면 직접 크롤링 (이용약관 위반 소지)
- 실제 발송(우편/이메일) 자동화
- 개별 기업에 대한 정식 진단·자문

## 4. 데이터 수집 상세

| API | 용도 | 비상장 외감법인 지원 여부 |
|---|---|---|
| 고유번호 API (corpCode.xml) | 전체 회사 고유번호 목록 | 가능 |
| 공시검색 API (list.json) | 외부감사보고서 등 공시 목록 조회 | 가능 |
| 기업개황 API (company.json) | 회사명·주소·대표자·전화번호·업종코드 조회 | 가능 |
| 재무제표 API (fnlttSinglAcntAll 등) | 매출·영업이익 등 구조화 수치 | **불가** — 상장법인 및 사업보고서 제출대상(IFRS) 비상장법인만 지원 |
| 공시서류원본 API (document.xml) | 감사보고서 원문 첨부파일 다운로드 | 가능 |

→ 3-2절의 재무정보는 **원문 첨부파일을 다운로드한 뒤 재무제표 표를 직접
파싱**하여 확보한다. 당기·전기 비교를 위해 재무상태표·손익계산서의 두 개
연도 컬럼을 함께 추출한다.

## 5. 처리 파이프라인 (조건 기반 재사용 가능한 구조)

1. 조건 입력: 지역, 매출액 범위, 업종(선택)
2. `corpCode.xml`로 전체 고유번호 목록 확보 (최초 1회 캐싱, 주기적 갱신)
3. `list.json`으로 대상 기간 내 "외부감사관련" 공시 목록 조회
4. 각 회사 `company.json` 호출 → 주소로 지역 필터링, 업종코드로 업종 필터링
5. 남은 회사에 대해 `document.xml`로 감사보고서 원문 다운로드
6. 원문에서 3-1, 3-2절 항목(당기·전기) 추출 (파싱)
7. 매출액 범위 조건으로 최종 필터링
8. 결과 CSV/Excel 저장
9. **(확장, 2026-07-15)** 8번까지 통과한 최종 결과 회사만 대상으로, 과거
   감사보고서를 추가로 조회해 최근 N년치(기본 4년) 재무 이력을 보강 —
   상세개발계획.md §4-6(STEP 7) 참고

## 6. 기술 스택 (제안)

- Python 3.x, `requests`(API 호출), `pandas`(데이터 처리)
- PDF 파싱: `pdfplumber` 또는 `PyMuPDF`
- HWP 파싱: `olefile` 기반 라이브러리 또는 한글 변환 도구 연계
- 조건 입력: CLI 인자 또는 간단한 설정파일(YAML/JSON)로 지역·매출액·업종 지정
- API 키는 환경변수로 관리, 코드에 하드코딩 금지

## 7. 리스크

- OpenDART API 일일 호출 한도 20,000건 — 조건에 따라 대상 모수가 커지면
  여러 날에 걸쳐 나눠 실행 필요
- 재무제표 원문 파싱은 회사별 서식 차이로 100% 자동화가 어려워 수동 검수 필요
- 이메일 주소는 확보 불가
- 매출액/업종 조건은 사후 필터링 구조라, 조건에 따라 파싱 대상 모수가
  커질 수 있음 (성능/시간에 영향)

## 8. 완료 기준 (Definition of Done) — Phase 1

- [ ] 지역/매출액/업종 조건을 입력받는 실행 구조 완성 (CLI 또는 설정파일)
- [ ] 조건에 맞는 회사 목록 자동 추출 스크립트 완성
- [ ] 회사별 기본정보(업체명/주소/전화번호/대표자/업종명/결산기준일/감사의견) 수집 완성
- [ ] 회사별 당기·전기 요약 재무정보(3-2절 전 항목) 파싱 완성
- [ ] 결과 CSV/Excel 산출 완성
- [ ] 샘플 10개사 수동 검수로 정확도 확인

## 9. 향후 논의 필요 사항 (Phase 2 착수 시)

- 전단지 최종 포맷 결정 (HTML/PDF/docx)
- 파생지표 산식 확정 (영업이익률, 순이익률, 유동비율, 부채비율, 매출액증가율 등)
- 업종별/규모별 코멘트 문구 세트 설계
- 업종 필터를 표준산업분류 대분류/중분류 중 어느 수준으로 지정할지
- 매출액 범위, 지역 단위(시/군 vs 광역) 입력 UX (CLI 옵션 vs 설정파일)
